## Purpose

These notes are:

- **Personal** — Written for myself as a learning aid
- **Informal** — Not polished documentation
- **Living** — Updated as I learn new things

Unlike ADRs (which document decisions), these notes capture knowledge and serve as a quick reference.

## Adding New Notes

When adding a new note file:

1. Create a descriptive filename (e.g., `debugging-tips.md`)
2. Add a brief entry to the table above
3. Keep notes focused on a single topic
