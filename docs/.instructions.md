# Documentation Instructions

Rules for files in the `docs/` directory.

## Stack

- **MkDocs Material** — config in `mkdocs.yml`, served with `hatch run docs:serve`.
- **MkDocs hooks** in `mkdocs-hooks/` — registered in `mkdocs.yml` under `hooks:`.
- **docs-build** CI workflow runs on all PRs (strict mode) and is in the CI gate.

## Conventions

- ADRs go in `docs/adr/` (see `docs/adr/.instructions.md` for detailed procedure).
- Design docs go in `docs/design/`.
- Reference material goes in `docs/reference/`.
- Use relative links between docs. The `repo_links.py` MkDocs hook rewrites repo-relative links to GitHub URLs at build time.
- Don't write marketing copy. State what something does, its limitations, and move on.

## Adding an MkDocs Hook

1. Create the hook in `mkdocs-hooks/<name>.py`.
2. Register it in `mkdocs.yml` under `hooks:`.
3. Update `mkdocs-hooks/README.md` inventory.

## Key Files

| File                              | Purpose                                   |
| --------------------------------- | ----------------------------------------- |
| `docs/workflows.md`              | Authoritative workflow inventory           |
| `docs/known-issues.md`           | Canonical issue tracker                    |
| `docs/design/architecture.md`    | System overview and CI/CD architecture     |
| `docs/design/tool-decisions.md`  | Tool comparison notes and rationale        |
| `docs/adr/README.md`             | ADR index                                  |
| `docs/reference/commands.md`     | Auto-generated command reference           |
| `docs/reference/template-inventory.md` | Template and example file inventory |
