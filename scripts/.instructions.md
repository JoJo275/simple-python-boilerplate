## Conventions (ADR 031)

### Structure

- Every script uses `argparse` for CLI with consistent flags: `--help`, `--version`, `--dry-run` (where applicable).
- Use `logging` for status/diagnostic messages (`logging.info()`, `logging.warning()`, `logging.error()`). Reserve `print()` only for primary output (tables, JSON, generated content).
- Configure logging in `main()` with `logging.basicConfig()`.
- Use `pathlib.Path` over `os.path` for all file operations.
- Use `subprocess.run()` with argument lists — never `shell=True`.
- Constants in `UPPER_SNAKE_CASE`.

### Shebang and Executable Bit

- Scripts with a shebang (`#!/usr/bin/env python3`) must be marked executable: `git add --chmod=+x scripts/my_script.py`.
- The `check-shebang-scripts-are-executable` pre-commit hook enforces this.

### Shared Modules

- Module names prefix with `_` to signal "internal library, not a CLI" — excluded from command reference generator.
- `_imports.py` — `import_sibling(name)` loads sibling scripts as modules. Use for cross-script imports:
  ```python
  from _imports import import_sibling
  _progress = import_sibling("_progress")
  ProgressBar = _progress.ProgressBar
  ```
- `_progress.py` — `ProgressBar` (determinate) and `Spinner` (indeterminate) with automatic Unicode/ASCII fallback. Import and use these instead of ad-hoc progress output.

## Python Style

- Absolute imports: `from simple_python_boilerplate.module import func`
- Type hints on all public functions and methods.
- Docstrings in Google style format.
- `tomllib` (stdlib 3.11+) for TOML reading — no third-party TOML lib.
- `importlib.metadata` for installed-package introspection.
- `shutil.get_terminal_size()` for terminal dimensions.
